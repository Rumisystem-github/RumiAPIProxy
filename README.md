# できること
クライアントとサーバー間でのデータ渡しを担当します。<BR>
例えば、クライアントがXML、サーバーがJSONを要求した場合に、XMLとJSONの相互変換を行います。<BR>
(まだXMLには対応してません。)<BR>
<BR>
また、データ圧縮にも対推しています。<BR>
現状は、無圧縮/ZSTDに対応しており、サーバーへは無圧縮として渡します。<BR>

# 使える圧縮
・Plain<BR>
無圧縮<BR>
・Zstd<BR>
Zstd<BR>

JSON<BR>

RSDF<BR>

# つかいかた(クライアント側)
`RSV-Accept-Encode`ヘッダーで「あなたはこのエンコードで送って」と指定します。<BR>
`RSV-Content-Encode`ヘッダーで、「こっちはこのエンコードで送ります」と指定します。<BR>
`Accept`ヘッダーで、「あなたはこのデータ形式で送って」と指定します。<BR>
`Content-Type`ヘッダーで「こっちはこのデータ形式で送ります」と指定します。<BR>
